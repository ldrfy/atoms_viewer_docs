# 性能与大规模数据

## 主要影响因素

原子数、帧数与可见图层数量是性能的核心因素。
导出倍率、抗锯齿与辅助显示也会增加负载。

## 内存与显存建议

多帧轨迹占用 CPU 内存，高分辨率导出占用显存。
建议删除不必要图层并控制同时可见对象数量。

## 面向百万原子的实践建议

- 优先单视图并减少叠加图层。
- 探索阶段降低原子半径或关闭键线。
- 仅在关键区间录制或导出。
- 定稿时再提高导出倍率。

## 页面无响应时

- 等待解析或重建完成，大文件可能需要更长时间。
- 刷新页面并减少图层数量后重试。
- 使用更强的 GPU 或拆分数据文件。
